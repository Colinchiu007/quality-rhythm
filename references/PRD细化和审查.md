

# 相关Skill

|Skill|作用|适合什么场景|
|---|---|---|
|`/pm`|需求审查、补充约束、标出模糊点|逐个 PRD 深度审查|
|`/spec`|模糊意图 → 结构化规格|把 PRD 转成可执行 spec|
|`/workflow-audit`|审计项目是否遵循 AGENTS.md 流程|检查流程规范|
|`/improve-codebase-architecture`|扫描代码库找架构深化机会|发现架构问题|

---

**如果用 `/pm` 来做，流程会是：**

1. 先告诉它 "审查这份 PRD，找出模糊点和缺失约束"
2. 它会主动追问边界条件、异常状态、接口契约
3. 最后输出一份带优先级的补充清单

---

要不要我用 `/pm` 重新跑一遍某份特定 PRD？比如：

- `TrendScope/docs/PRD.md` — 热榜那个问题最多
- `shared-models/docs/PRD.md` — 契约层是全局关键

# 这些 skill 覆盖了从需求到交付的完整生命周期：

---

## 需求阶段

**`/pm`** — 模糊想法 → 专业 PRD

- 使用场景：有产品想法但还没想清楚，需要有人追问边界条件、异常状态、交互流程
- 典型触发："我有个想法但还没想清楚"、"帮我梳理需求"
- 产出：完整 PRD 文档，含约束、交互流程、错误状态

**`/spec`** — 可执行规格（5 阶段：Why → Scope → Technical → Draft → File）

- 使用场景：PRD 写好了但还不够精确，需要转成开发者能直接用的技术规格
- 典型触发："这个需求怎么落地"、"帮我写技术规格"
- 产出：结构化技术规格文档

**`/define-goal`** — 定义可量化目标

- 使用场景：目标模糊，需要明确成功标准和验证方式
- 典型触发："我想优化性能"、"帮我定个目标"
- 产出：具体、可衡量、有边界的任务目标

---

## 规划阶段

**`/create-plan`** — 创建方案文档骨架

- 使用场景：需求清楚了，开始写实施计划
- 典型触发："开始做方案"、"帮我起草一个 plan"
- 产出：方案文档模板，后续可接迭代审查

**`/workflow-audit`** — 审计项目流程

- 使用场景：想检查项目是否遵循 AGENTS.md 规定的开发流程
- 典型触发："看看我们的流程有没有问题"、"审计一下"
- 产出：流程合规报告

---

## 开发阶段

**`/improve-codebase-architecture`** — 架构深化机会

- 使用场景：项目跑了一段时间，想看看有没有架构优化空间
- 典型触发："代码有点乱了"、"有没有架构问题"
- 产出：架构问题报告 + 改进建议

**`/auto-exec`** — 自主编排执行

- 使用场景：目标清楚了，让 AI 自动分解任务并跨会话批量执行
- 典型触发："目标定了，开干"
- 产出：任务分解 + 自动执行

---

## 一句话总结

```
有想法 → /pm 梳理
想清楚 → /spec 规格化
定目标 → /define-goal
做方案 → /create-plan
搞开发 → /auto-exec
检查 → /workflow-audit + /improve-codebase-architecture
```
